## 🚧 Blockers — ทำก่อนเลย

| งาน | ทำไมสำคัญ |
|---|---|
| ติดตั้ง SSL บน `planeatdev.duckdns.org` | LINE Webhook ต้องการ HTTPS — ทดสอบ approval จริงไม่ได้ถ้าไม่มี |
| ทดสอบ LINE approval จริง | Postback + Status Card + Carousel พร้อมแล้ว รอแค่ SSL |
| แก้ Dev DB แยกออกจาก Prod | ตอนนี้ dev ใช้ DB `planeat` เดียวกับ prod — อันตราย ต้องตั้ง `MONGO_DB` env ใหม่ |

## 🔵 Phase 2 — Sales Pipeline (Q2/2026)

**เป้าหมาย**: ปิดดีลเร็วขึ้น ไม่พลาด Lead

### มีแล้ว ✅
- `routers/deals.py` + `models/deal.py` — CRUD deals ครบ
- `routers/activities.py` + `models/activity.py` — Activity Log
- `/sales/deals` frontend page

### ยังต้องทำ

**1. Kanban Board Deals**
- หน้า `/deals` แสดง deals แบบ kanban (Lead → Qualified → Proposal → Won/Lost)
- drag & drop ย้าย stage
- เชื่อมกับ CRM ลูกค้า

**2. Activity Log**
- บันทึก touchpoint ทุกครั้งที่ contact ลูกค้า (โทร, ส่งอีเมล, ประชุม)
- แสดงใน timeline ต่อ deal

**3. LINE Follow-up Reminder**
- ตั้ง reminder ผ่านระบบ → แจ้งพนักงานทาง LINE เมื่อถึงเวลา
- ARQ Worker + LINE push ส่วนตัว

**4. Sales Dashboard**
- Win Rate, Pipeline Value, ยอดแต่ละ sales
- เชื่อมกับ [[LINE Command Center]] — พิมพ์ "ตัวชี้วัดทีมขาย" ใน LINE ดูได้เลย

## 📱 LINE Command Center — รายละเอียด

ผู้บริหารไม่ต้องเปิดคอมพิวเตอร์ — สั่งงานผ่าน LINE ได้ทันที

| คำสั่ง | ผลลัพธ์ |
|---|---|
| `สรุปวันนี้` | ยอดขาย + ค่าใช้จ่าย + กำไรสุทธิ |
| `สรุปเดือนนี้` | ภาพรวมการเงินรายเดือน |
| `ดีลรออนุมัติ` | รายการที่รอผู้บริหาร |
| `สต๊อกวิกฤต` | สินค้าใกล้หมดคลัง |
| `ลูกค้าใหม่วันนี้` | รายชื่อลูกค้าใหม่ |
| `ตัวชี้วัดทีมขาย` | Win Rate, Pipeline, ยอดแต่ละเซลส์ |
| `✅ [รหัส]` | อนุมัติรายการ |
| `❌ [รหัส]` | ปฏิเสธรายการ |

> ได้เปรียบเพราะทุกคนใช้ LINE อยู่แล้ว — Zero Learning Curve
